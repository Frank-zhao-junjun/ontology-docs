# 工作流（Workstreams）与实施计划对齐

对应《实施计划》里程碑，建议长期并行 **5 条工作流**（可由子代理分别主责草稿，PM 合并）。

| 工作流 | 范围 | 主要对齐文档 | 典型里程碑 |
| --- | --- | --- | --- |
| **WS1 元模型与生成** | 五类元模型、发布、快照、生成器占位 | 需求 §4、HLD §4.5、LLD §3 | M1–M2 |
| **WS2 编排与工具** | AI 编排器、工具路由、自愈、对话 API | 需求 §5、7、HLD §4.3、LLD §4 | M3 |
| **WS3 规则·状态机·事件** | 规则引擎、状态机、进程内事件、技能联动 | 需求 §4.3–4.6、§6、LLD §5 | M3 |
| **WS4 交互与网关** | 三栏 UI、动作协议、API 网关、认证授权 | 需求 §7、§2.4、HLD §4.1–4.2 | M4 |
| **WS5 工程与质量** | 仓库骨架、CI、RTM、可观测、安全基线 | 实施计划 M1、HLD §9–12 | M1 起持续 |

## 与「子代理」分工示例

- **一次迭代**可并行：WS1 起草 Schema 草案 + WS5 起草目录与 CI 占位 + WS2 起草 API 契约表。  
- **合并点**：PM 把结论写入 `tracking/` 与 `sprints/`，再进入编码仓库。
