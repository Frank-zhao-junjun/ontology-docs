# PM 编排说明（与子代理协作）

## 能否「多 sub-agent 分工」？

**可以。** 在本环境中通过 **并行子任务（Task）** 实现：不同子任务各自阅读指定文档、起草设计或清单，**产出文件**统一落在 `D:\AI\Ontology\Cursor\`（或你指定的代码仓库路径）。

注意：

- 子任务**不替代**你对优先级与验收的最终决策。
- 同一文件的编辑应 **串行合并**，避免冲突。
- 需求/架构/LLD **正式稿**仍以 `D:\AI\Ontology\` 根下文件为准；本目录多为 **追踪、清单、提案**。

## 推荐调用方式

你对主会话说清即可，例如：

- 「按 WS1+WS5 并行：一人扩 RTM 到条款级，一人写 `repo-layout` 初稿」
- 「对照 `开发Backlog.md`，把 M1 能从 Backlog 里拆出的任务写到 `sprints/M1-tasks.md`」
