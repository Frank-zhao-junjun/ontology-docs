# EPC信息对象建模规则

## 1. 定位

EPC 信息对象是“流程语境中的业务信息载体”，用于描述某个业务活动的输入、输出和流转对象。

EPC 信息对象不是新的数据真值层，不承担实体结构定义、字段标准定义或主数据记录维护职责。

## 2. 与现有模型的边界

### 2.1 数据模型

- 回答系统中“有什么业务对象”
- 负责实体、属性、关系、聚合边界定义
- 是结构真值

### 2.2 元数据

- 回答字段“是什么意思”
- 负责字段中文名、英文名、类型、标准、值域等定义
- 是字段语义真值

### 2.3 主数据

- 回答“标准参考对象来自哪里”
- 负责供应商、客户、员工、物料等稳定参考对象及其记录
- 是参考数据真值

### 2.4 EPC信息对象

- 回答“流程里哪些业务信息在流转”
- 负责活动输入/输出的信息载体表达
- 是流程视角下的业务对象投影

## 3. EPC信息对象来源类型

### 3.1 `aggregate`

- 来源于聚合根实体
- 自动派生
- 名称、来源引用、属性摘要不可手工改写

### 3.2 `child_entity`

- 来源于聚合内子实体
- 自动派生
- 名称、来源引用、属性摘要不可手工改写

### 3.3 `masterdata`

- 来源于实体属性引用的主数据
- 自动派生
- 名称、来源引用不可手工改写

### 3.4 `manual`

- 来源于流程补充语义
- 仅用于当前四大元模型暂时无法稳定推导的信息载体
- 允许手工新增、编辑、删除

## 4. 编辑规则

### 4.1 自动派生对象

对 `aggregate` / `child_entity` / `masterdata` 对象：

- 允许编辑：流程说明 `description`
- 禁止编辑：`name`、`sourceType`、`sourceRefId`、`attributes`
- 禁止删除

### 4.2 手工补充对象

对 `manual` 对象：

- 允许编辑：`name`、`attributes`、`description`
- 允许删除
- `sourceType` 固定为 `manual`

## 5. 重生成规则

### 5.1 自动派生对象刷新

- 每次重生成时，系统基于最新四大元模型重新派生 `aggregate` / `child_entity` / `masterdata` 信息对象
- 自动刷新这些对象的名称、属性摘要与引用来源

### 5.2 手工对象保留

- `manual` 信息对象在重生成时必须保留
- 不允许因重生成而丢失手工对象

### 5.3 派生对象的流程说明保留

- 若用户为自动派生对象补充了 `description`，则重生成时保留该说明
- 但结构字段仍以源模型重新计算为准

## 6. 校验规则

### 6.1 基础规则

- `manual` 信息对象名称不能为空
- `manual` 信息对象不得与同一 EPC profile 中已有对象重名
- 自动派生对象必须带有合法 `sourceRefId`

### 6.2 真值保护规则

- EPC 信息对象不得反向覆盖数据模型、元数据或主数据定义
- EPC 页面内不得直接编辑派生对象的结构属性

## 7. 产品规则

1. 能从数据模型派生的，不允许在 EPC 内重复建真值。
2. 能从主数据引用推导的，不允许在 EPC 内重新录入主数据对象。
3. 元数据只属于字段语义层，不在 EPC 中作为平级信息对象维护。
4. EPC 仅补充流程语义，不篡改主建模层。
