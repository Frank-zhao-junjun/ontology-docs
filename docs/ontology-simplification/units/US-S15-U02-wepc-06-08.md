## 1. 目标（一句话）

在 `lintBusinessEpc` 中增加 W-EPC-06、W-EPC-07、W-EPC-08 三条新校验规则。

## 2. 范围

### In Scope

- W-EPC-06：EPC 步骤引用的 elementName 与 MetaElement.name 不一致时警告
- W-EPC-07：EPC 步骤引用的 dimension 与 MetaElement.dimension 不一致时警告
- W-EPC-08：已确认 EPC 没有任何行为（E2）要素关联时警告
- 对应单元测试

### Out of Scope

- W-EPC-09~17（后续 Unit）
- WarningCenter UI 适配（U06 统一做）

## 3. 技术设计

### EpcStepElementRef 增加 elementName 字段

在 `src/types/ontology.ts` 的 `EpcStepElementRef` 接口增加可选字段：

```typescript
export interface EpcStepElementRef {
  dimension: MetaDimension;
  elementId: string;
  versionPin: VersionPin;
  inlineNew?: boolean;
  inlinePayload?: unknown;
  elementName?: string;  // ← 新增：引用时缓存的要素 name，用于 W-EPC-06 对比
}
```

理由：`elementName` 记录的是引用建立时要素的名称快照，与当前 `MetaElement.name` 对比可检测名称变更。

### W-EPC-06：名称不一致

**触发条件**：在已确认 EPC 的步骤中，`step.elementRef.elementName` 存在且不等于当前 `meta.name`

```typescript
// 在 inspectConfirmedEpcStep 中增加
if (step.elementRef?.elementName && meta && step.elementRef.elementName !== meta.name) {
  pushWarning(warnings, {
    ruleId: 'W-EPC-06',
    message: `步骤「${step.name}」引用要素名称已变更：缓存名「${step.elementRef.elementName}」→ 当前名「${meta.name}」`,
    moduleKind: 'EPC',
    moduleId: epc.id,
    epcId: epc.id,
    stepId: step.id,
    elementId: step.elementRef.elementId,
  });
}
```

**不做**：
- 如果 `elementName` 为 undefined → 不触发 W-EPC-06（兼容旧引用）
- 如果 meta 不存在 → 已由 W-EPC-05 覆盖

### W-EPC-07：维度不一致

**触发条件**：`step.elementRef.dimension` ≠ `meta.dimension`

```typescript
if (meta && step.elementRef && step.elementRef.dimension !== meta.dimension) {
  pushWarning(warnings, {
    ruleId: 'W-EPC-07',
    message: `步骤「${step.name}」引用的维度不一致：ref dim=${step.elementRef.dimension} ≠ actual dim=${meta.dimension}`,
    moduleKind: 'EPC',
    moduleId: epc.id,
    epcId: epc.id,
    stepId: step.id,
    elementId: step.elementRef.elementId,
  });
}
```

### W-EPC-08：行为密度（无 E2）

在 `getConfirmedEpcSnapshots` 循环中，对每个已确认 EPC 扫描全部步骤：

```typescript
// 在 lintBusinessEpc 中增加 EPC 级扫描
for (const epc of getConfirmedEpcSnapshots(records, epcProcesses)) {
  const hasE2Step = (epc.steps ?? []).some(s => s.elementRef?.dimension === 'E2');
  if (!hasE2Step) {
    pushWarning(warnings, {
      ruleId: 'W-EPC-08',
      message: `流程「${epc.name}」没有任何行为（E2）要素引用`,
      moduleKind: 'EPC',
      moduleId: epc.id,
      epcId: epc.id,
    });
  }
}
```

备注：W-EPC-08 是 **EPC 级别**的规则，与 W-EPC-01~05/06/07（step 级别）不同。
