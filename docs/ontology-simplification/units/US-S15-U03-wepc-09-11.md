# US-S15-U03：W-EPC-09~11 规则（数据绑定 + 实体绑定 + 角色绑定）

| 字段 | 值 |
|------|-----|
| **所属 US** | [US-S15](../us/US-S15-epc-wepc-ext.md) |
| **状态** | 草稿 |
| **依赖** | U01（类型已扩展），U02（linter 函数模式已建立） |
| **预估文件** | `src/lib/business-epc-linter/index.ts`, `tests/unit/business-epc-linter.spec.ts` |

## 1. 目标（一句话）

在 `lintBusinessEpc` 中增加 W-EPC-09、W-EPC-10、W-EPC-11 三条新校验规则，检查要素引用的数据绑定、实体绑定和角色绑定完整性。

## 2. 范围

### In Scope

- **W-EPC-09**：EPC 步骤引用了 E1（数据模型）要素，但该要素缺少数据源/实体绑定（`entityId` 未设置）→ 警告
- **W-EPC-10**：EPC 步骤引用了要素，但该要素未绑定到具体 Entity（`entityId` 未设置）→ 警告
- **W-EPC-11**：EPC 步骤引用了要素，但该要素未关联治理角色/权限（E5 时检查 `hasPolicy`）→ 警告
- 对应单元测试

### Out of Scope

- W-EPC-12~17（后续 Unit）
- WarningCenter UI（U06）

## 3. 技术设计

所有规则均作用于 `inspectConfirmedEpcStep` 中（step 级别），在已确认 EPC 的步骤循环内检查。

### W-EPC-09：E1 数据依赖

当步骤引用 `dimension === 'E1'` 的要素时，检查要素是否有 `entityId`：

```typescript
// 在 inspectConfirmedEpcStep 中，当 meta 存在时
if (step.elementRef?.dimension === 'E1' && meta && !meta.entityId) {
  pushWarning(warnings, {
    ruleId: 'W-EPC-09',
    message: `步骤「${step.name}」引用 E1 要素「${meta.name}」但缺少数据源/实体绑定`,
    moduleKind: 'EPC',
    moduleId: epc.id,
    epcId: epc.id,
    stepId: step.id,
    elementId: step.elementRef.elementId,
  });
}
```

### W-EPC-10：实体绑定

当步骤引用了任意维度的要素时，检查该要素是否有 `entityId`。与 W-EPC-09 的区别：W-EPC-09 仅 E1，W-EPC-10 覆盖所有维度（但排除 E1 检查项以避免重复警告，或保持两条独立——由 TDD 测试确定行为）。

```typescript
// 在 inspectConfirmedEpcStep 中，当 meta 存在且 dimension !== 'E1' 时（避免与 W-EPC-09 重复）
if (meta && step.elementRef?.dimension !== 'E1' && !meta.entityId) {
  pushWarning(warnings, {
    ruleId: 'W-EPC-10',
    message: `步骤「${step.name}」引用要素「${meta.name}」但未绑定到具体 Entity`,
    moduleKind: 'EPC',
    moduleId: epc.id,
    epcId: epc.id,
    stepId: step.id,
    elementId: step.elementRef.elementId,
  });
}
```

### W-EPC-11：角色绑定

当步骤引用了 E5（岗位角色）要素时，检查该要素是否有关联的治理角色/权限定义（`hasPolicy === true`）：

```typescript
// 在 inspectConfirmedEpcStep 中
if (step.elementRef?.dimension === 'E5' && meta && !meta.hasPolicy) {
  pushWarning(warnings, {
    ruleId: 'W-EPC-11',
    message: `步骤「${step.name}」引用 E5 要素「${meta.name}」但未关联治理角色/权限`,
    moduleKind: 'EPC',
    moduleId: epc.id,
    epcId: epc.id,
    stepId: step.id,
    elementId: step.elementRef.elementId,
  });
}
```
