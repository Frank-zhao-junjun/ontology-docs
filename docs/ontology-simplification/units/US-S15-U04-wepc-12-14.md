# US-S15-U04：W-EPC-12~14 规则（事件起止 + 行为/事件绑定）

| 字段 | 值 |
|------|-----|
| **所属 US** | [US-S15](../us/US-S15-epc-wepc-ext.md) |
| **状态** | 草稿 |
| **依赖** | U01（类型字段），U02（linter 模式），U03（step 级规则模式） |
| **预估文件** | `src/lib/business-epc-linter/index.ts`, `tests/unit/business-epc-linter.spec.ts` |

## 1. 目标（一句话）

在 `lintBusinessEpc` 中增加 W-EPC-12、W-EPC-13、W-EPC-14 三条规则，检查 EPC 事件起止和要素绑定完整性。

## 2. 范围

### In Scope

- **W-EPC-12**：EPC 首尾步骤均未引用 E3（事件）要素 → 警告（EPC 级别）
- **W-EPC-13**：E2（行为）步骤引用的要素缺少 `stateMachineId` → 警告（step 级别）
- **W-EPC-14**：E3（事件）步骤引用的要素缺少 `eventId` → 警告（step 级别）
- 对应单元测试

### Out of Scope

- W-EPC-15~17（U05）
- WarningCenter UI（U06）

## 3. 技术设计

### W-EPC-12：事件起止（EPC 级别）

在 `getConfirmedEpcSnapshots` 循环中，对每个已确认 EPC：

```typescript
if (steps.length > 0) {
  const firstDim = steps[0].elementRef?.dimension;
  const lastDim = steps[steps.length - 1].elementRef?.dimension;
  if (firstDim !== 'E3' && lastDim !== 'E3') {
    pushWarning(warnings, {
      ruleId: 'W-EPC-12',
      message: `EPC「${epc.name}」首尾步骤均未引用事件要素（E3 维度），缺少事件起点或终点`,
      moduleKind: 'EPC',
      moduleId: epc.id,
      epcId: epc.id,
    });
  }
}
```

### W-EPC-13：行为状态机绑定（step 级别）

在 `inspectConfirmedEpcStep` 中：

```typescript
if (step.elementRef.dimension === 'E2' && meta && !meta.stateMachineId) {
  pushWarning(warnings, {
    ruleId: 'W-EPC-13',
    message: `步骤「${step.name}」引用的行为要素「${meta.name}」未绑定状态机`,
    moduleKind: 'EPC',
    moduleId: epc.id,
    epcId: epc.id,
    stepId: step.id,
    elementId: step.elementRef.elementId,
  });
}
```

### W-EPC-14：事件定义绑定（step 级别）

在 `inspectConfirmedEpcStep` 中：

```typescript
if (step.elementRef.dimension === 'E3' && meta && !meta.eventId) {
  pushWarning(warnings, {
    ruleId: 'W-EPC-14',
    message: `步骤「${step.name}」引用的事件要素「${meta.name}」未绑定事件定义`,
    moduleKind: 'EPC',
    moduleId: epc.id,
    epcId: epc.id,
    stepId: step.id,
    elementId: step.elementRef.elementId,
  });
}
```
