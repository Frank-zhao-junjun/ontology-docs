# Ontology 本体模型建模工具 - 功能实现说明

> 本文档记录 Ontology 本体模型可视化建模工具的所有已实现功能，与 README.md 保持同步。

## 项目概述

基于 Next.js 16 的本体模型可视化建模工具，支持任意领域及其五大元模型（数据、行为、规则、流程、事件）的可视化建模，并输出完整的建模手册。

---

## 一、领域建模

### 1.1 领域选择 ✅

**功能描述**：内置8大行业领域模板，支持快速选择建模所属领域。

**已实现领域**：
| 领域ID | 中文名称 | 英文名称 | 图标 | 颜色 |
|--------|----------|----------|------|------|
| discrete_manufacturing | 离散制造 | Discrete Manufacturing | 🏭 | #3B82F6 |
| process_manufacturing | 流程制造 | Process Manufacturing | 🧪 | #10B981 |
| retail_ecommerce | 零售电商 | Retail & E-commerce | 🛒 | #F59E0B |
| financial_services | 金融服务 | Financial Services | 💰 | #EF4444 |
| healthcare | 医疗健康 | Healthcare | 🏥 | #8B5CF6 |
| education_training | 教育培训 | Education & Training | 📚 | #06B6D4 |
| logistics_supply_chain | 物流供应链 | Logistics & Supply Chain | 🚚 | #EC4899 |
| energy_environment | 能源环保 | Energy & Environment | 🌿 | #22C55E |

**预设领域示例**：
- 合同管理、客户关系、库存管理、人力资源、财务管理、项目管理等

### 1.2 项目创建 ✅

**功能描述**：基于领域创建建模项目，支持项目名称、描述配置。

**实现细节**：
- 项目名称：中文名称
- 项目描述：业务场景说明
- 领域关联：自动关联选择的领域
- 持久化：自动保存到 localStorage

### 1.3 实体分组管理 ✅

**功能描述**：支持将实体按项目/模块分组管理。

**实现细节**：
- 项目列表：左侧展示项目分组
- 实体归属：每个实体归属于一个项目
- 全部视图：支持查看所有项目的实体
- 项目CRUD：支持新建、编辑、删除项目分组

## 二、五大元模型编辑

### 2.1 数据模型 ✅

**功能描述**：实体属性定义、关系定义。

**属性定义**：
| 属性 | 说明 | 可选值 |
|------|------|--------|
| 名称 | 中文名称 | 自定义 |
| 英文名 | 英文名称 | 大写字母+下划线 |
| 类型 | 数据类型 | string, text, integer, decimal, boolean, date, datetime, enum, reference |
| 必填 | 是否必填 | true/false |
| 唯一 | 是否唯一 | true/false |
| 描述 | 业务说明 | 自定义 |

**关系定义**：
| 关系类型 | 说明 |
|----------|------|
| one_to_one | 一对一 |
| one_to_many | 一对多 |
| many_to_many | 多对多 |

### 2.2 行为模型 ✅

**功能描述**：状态机设计，支持状态定义、转换规则、触发器配置。

**状态机定义**：
- 状态机名称
- 关联实体
- 状态字段

**状态定义**：
| 属性 | 说明 |
|------|------|
| 名称 | 状态名称 |
| 初始状态 | 是否为初始状态 |
| 终止状态 | 是否为终止状态 |
| 描述 | 状态说明 |

**状态转换**：
| 属性 | 说明 | 可选值 |
|------|------|--------|
| 名称 | 转换名称 | 自定义 |
| 起始状态 | 从哪个状态 | 状态列表 |
| 目标状态 | 到哪个状态 | 状态列表 |
| 触发方式 | 触发类型 | manual, automatic, scheduled |
| 描述 | 转换说明 | 自定义 |

### 2.3 规则模型 ✅

**功能描述**：字段验证、跨字段验证、业务约束规则。

**规则类型**：
| 类型 | 说明 |
|------|------|
| field_validation | 字段级校验 |
| cross_field_validation | 跨字段校验 |
| cross_entity_validation | 跨实体校验 |
| aggregate_validation | 聚合校验 |
| temporal_rule | 时序规则 |

**规则属性**：
- 规则名称、类型
- 关联实体、字段
- 条件表达式
- 错误消息、严重程度（error/warning/info）

### 2.4 流程模型 ✅

**功能描述**：业务流程编排，支持多种步骤类型。

**流程编排**：
- 流程名称、描述
- 入口点定义
- 步骤序列

**步骤类型**：
| 类型 | 说明 |
|------|------|
| intent_clarification | 意图澄清 |
| query_generation | 查询生成 |
| data_retrieval | 数据检索 |
| validation | 验证 |
| skill_execution | 技能执行 |
| insight_generation | 洞察生成 |
| visualization | 可视化 |
| presentation | 展示 |
| decision | 决策 |
| notification | 通知 |

### 2.5 事件模型 ✅

**功能描述**：事件定义、事件订阅、触发器配置。

**事件定义**：
| 属性 | 说明 | 可选值 |
|------|------|--------|
| 名称 | 事件名称 | 自定义 |
| 关联实体 | 所属实体 | 实体列表 |
| 触发时机 | 触发类型 | create, update, delete, state_change, custom |
| 条件 | 触发条件 | 表达式 |
| 载荷 | 输出字段 | 字段列表 |

**事件订阅**：
| 属性 | 说明 | 可选值 |
|------|------|--------|
| 名称 | 订阅名称 | 自定义 |
| 关联事件 | 订阅的事件 | 事件列表 |
| 处理方式 | 同步/异步 | sync, async |
| 动作类型 | 执行动作 | skill, webhook, notification, script |
| 动作引用 | 动作地址 | URL/ID |

## 三、元数据管理

### 3.1 Excel初始化 ✅

**功能描述**：从预置Excel导入57条标准元数据字段。

**数据来源**：`assets/元数据清单.xlsx`

**领域分布**：

| 仓储 | 4 |
| 技术/研发 | 4 |
| 财务 | 3 |
| 库存 | 1 |

### 3.2 CRUD操作 ✅

**功能描述**：支持元数据的增删改查。

**操作支持**：
- 列表展示：表格形式展示所有元数据
- 搜索过滤：按领域、名称、英文名、描述搜索
- 新增：添加自定义元数据
- 编辑：修改元数据信息
- 删除：移除元数据

### 3.3 AI优先匹配 ✅

**功能描述**：AI生成属性时优先从元数据列表匹配。

**实现逻辑**：
1. AI生成请求时传递元数据列表
2. 提示词要求AI优先匹配元数据定义
3. 保持命名、类型、值范围一致性

### 3.4 元数据类型定义 ✅

```typescript
interface Metadata {
  id: string;
  domain: string;         // 领域（如：财务、物料、生产等）
  name: string;           // 字段中文名
  nameEn: string;         // 字段英文名
  description: string;    // 业务含义
  type: string;           // 字段属性/类型
  valueRange?: string;    // 值范围
  standard?: string;      // 参考标准
  source?: string;        // 信息源头
  createdAt: string;
  updatedAt: string;
}
```

## 四、AI智能生成

### 4.1 模型建议 ✅

**功能描述**：基于实体和领域信息，AI自动生成五大模型建议。

**生成内容**：
- 数据模型建议（属性、关系）
- 行为模型建议（状态、转换）
- 规则模型建议（验证规则）
- 流程模型建议（业务流程）
- 事件模型建议（事件、订阅）

### 4.2 一键应用 ✅

**功能描述**：可将AI建议一键应用到当前实体。

**应用逻辑**：
| 类型 | 操作 |
|------|------|
| 属性 | 添加到实体属性列表 |
| 关系 | 添加到实体关系列表 |
| 状态机 | 创建新状态机 |
| 规则 | 添加到规则列表 |
| 事件 | 添加到事件定义列表 |
| 订阅 | 关联事件并添加订阅 |

### 4.3 API接口 ✅

```
POST /api/generate-model
```

**请求体**：
```json
{
  "entity": { "name": "物料", "nameEn": "Material", ... },
  "domain": { "name": "离散制造", ... },
  "project": { "name": "生产管理", ... },
  "existingModels": { ... },
  "metadataList": [ ... ]
}
```

**返回格式**：
```json
{
  "success": true,
  "data": {
    "dataModel": { "suggestedAttributes": [...], "suggestedRelations": [...] },
    "behaviorModel": { ... },
    "ruleModel": { ... },
    "processModel": { ... },
    "eventModel": { ... }
  }
}
```

|------|------|

## 六、技术实现

### 6.1 技术栈

| 技术 | 版本 | 说明 |
|------|------|------|
| Next.js | 16.1.1 | App Router |
| React | 19.2.3 | 核心框架 |

### 6.2 核心文件

| 文件路径 | 说明 |
|----------|------|
| `src/types/ontology.ts` | 类型定义 |
| `src/store/ontology-store.ts` | Zustand状态管理 |
