## 四、后端 API 功能清单（单测已覆盖或手工可补验）

| 模块 | 能力 | 路由/方法 | 验证要点 |
| --- | --- | --- | --- |
| 健康 | 存活探针 | `GET /health` | `200`，`status` 为 ok |
| 元模型 | 结构校验 | `POST /api/meta-models/validate` | 返回 `is_valid` / `errors` / `warnings` |
| 元模型 | 草稿定义 | `POST /api/meta-models/definitions` | 缺参时 `400` + `error_code`；合法时写入草稿 |
| 元模型 | 发布 | `POST /api/meta-models/publish` | 发布链、运行时索引重建（与单测一致） |
| 元模型 | 当前快照 | `GET /api/meta-models/snapshots/current` | 可读到当前发布快照 |
| 元模型 | 回滚 | `POST /api/meta-models/rollback` | 与单测场景一致 |
| 领域 | 统一写路径 | `POST /api/domain/validate-and-transition` | 规则 → 状态机 → 事件（桩/最小实现） |
| 对话 | 执行 | `POST /api/chat/execute` | `assistant_message`、`orchestration`、`trace_id`；分析意图含 **`tool_results`**、**`RENDER_CHART`** |
| 对话 | 重试 | `POST /api/chat/retry` | 超限 `A4092` |
| 对话 | 会话上下文 | `GET /api/chat/session/<id>/context` | 返回会话侧状态摘要 |
| 运行时 | 规则刷新 | `POST /api/runtime/rules/refresh` | 热刷新路径（见 Sprint C 文档） |
| 运行时 | 异步事件 | `POST /api/runtime/events/dispatch-async` | `202` 或约定行为 |
| 横切 | CORS / Trace | 全局 | `X-Trace-Id`；CORS 配置与前端开发代理匹配 |
