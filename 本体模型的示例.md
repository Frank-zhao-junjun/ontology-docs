## 1. 语义理解层

当前代码只有 `entities` + `intent` + `suggestedActions`，缺少 DDD 分层建模。

| 要素 | 说明 | 代码中的体现 |
|------|------|------------|
| **业务域** | 顶层业务划分 | `manufacturing` / `customer-service` / `supply-chain` / `general` |
| **业务场景** | 域内业务上下文 | 当前缺失，代码中隐含在 intent 描述里 |
| **子域** | 场景下的细分领域 | 当前缺失 |

**制造业子域**：
- 产线管理场景
- 设备管理场景
- 质量管理场景
- 产能分析场景

**客服子域**：
- 客户咨询场景
- 工单处理场景
- 投诉升级场景
- 知识查询场景

**供应链子域**：
- 库存管理场景
- 物流调度场景
- 供应商管理场景
- 采购管理场景

| 要素 | 说明 | 代码中的体现 |
|------|------|------------|
| **聚合根** | 一致性边界内的根实体 | 当前缺失概念 |
| **实体** | 有唯一标识的业务对象 | `productionLines` / `equipment` / `customers` / `tickets` / `inventoryItems` 等 |
| **值对象** | 无唯一标识的属性组合 | 当前散落在实体字段中，未提炼 |
| **领域关系** | 实体间的关联类型 | 当前缺失，仅靠字段引用（如 `equipment.lineId`） |
| **关系类型** | 组合/聚合/依赖/关联 | 当前缺失 |

**聚合根**（应补充）：

| 业务域 | 聚合根 | 聚合内实体/值对象 |

| 制造业 | 产线 (ProductionLine) | 设备(归属产线)、产能指标、效率值对象 |

| 供应链 | 仓库 (Warehouse) | 库存项(归属仓库)、利用率值对象 |
| 供应链 | 采购单 (PurchaseOrder) | 采购项值对象、供应商引用 |
| 供应链 | 物流单 (Shipment) | 路线值对象、延迟状态值对象 |

**值对象**（应提炼）：

| 值对象 | 字段 | 所属聚合根 |

| 工单状态 | open / in_progress / resolved / closed | Ticket |
| 订单状态 | pending / confirmed / shipped / delivered | Order |
| 物流状态 | pending / in_transit / delivered / delayed | Shipment |
| 采购状态 | pending / confirmed / completed / cancelled | PurchaseOrder |
| 合规状态 | compliant / pending_review / non_compliant | Supplier |

| SLA | targetHours / elapsedHours / isBreached | Ticket |
| 质量指标 | yieldRate / defectRate / complaintRate / standard / isPassing | Product |

**领域关系**（应补充）：

| 关系 | 类型 | 说明 |
|------|------|------|
| 设备 → 产线 | 聚合（属于） | equipment.lineId → productionLine.id |

| 订单 → 客户 | 聚合（属于） | order.customerId → customer.id |

| 采购单 → 供应商 | 关联（采购自） | purchaseOrder.supplierId → supplier.id |

## 2. 行为执行层

当前代码只有 `ontology_plan_execute` 一个工具，内部按 `task_type` 分发。缺少以下要素：

| 要素 | 说明 | 代码现状 |
|------|------|---------|
| **EPC 流程** | 事件-过程-链，定义流程步骤 | 完全缺失 |
| **行为** | 原子操作单元 | 隐含在 task_type 分支中，未显式定义 |
| **行为规则** | 何时触发何种行为 | 散落在 index.ts 的 if/switch 中 |
| **校验规则** | 前置/后置条件 | 缺失 |
| **状态机** | 实体状态转移定义 | 状态值硬编码在 data.ts 中，无转移规则 |
| **指标定义** | 量化评估标准 | 数据中有值，缺定义（目标/阈值/计算公式） |

**EPC 流程**（应补充）：

| 业务域 | 流程名称 | 步骤链 |

| 供应链 | 物流异常流程 | 延迟事件 → 影响评估 → 客户通知 → 备选路线 |

| 工单SLA检查 | 客服 | ticketId | SLA剩余/是否违约 |
| 知识库检索 | 客服 | query | FAQ匹配结果 |

| 风险影响评估 | 供应链 | riskId | 影响范围+建议措施 |

**状态机**（应补充转移规则）：

| 实体 | 状态转移 | 触发条件 |
|------|---------|---------|

**应补充的要素**：

| 要素 | 说明 |
|------|------|

| **护栏 (Guardrail)** | 安全/合规红线，触发即阻断 |
